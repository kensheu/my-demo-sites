# My Demo Sites

展示用測試頁面，整合三個系統的統一導覽介面。

## 線上網址

https://kensheu.github.io/my-demo-sites/

## 系統架構

全站透過 `shared-nav.js` / `shared-nav.css` 注入統一頂部列與左側邊欄，支援三個系統切換：

| 系統 | 說明 | 頁面 |
|------|------|------|
| 內部後台 | 採購／管理人員使用 | 首頁儀表板、公告管理、餐點管理、餐點異動紀錄、店家異動審核、菜單排程管理、菜單發布管理、菜單管理異動紀錄、菜單模板管理、訂單管理、客訴管理、帳號管理、問題單&ECR、點數 & 折價券管理 |
| 店家後台 | 餐廳店家使用 | 公告通知（含排程通知）、餐點管理、訂單明細 |
| 訂餐前台 | 訂餐會員使用 | 公告通知（含據點切換）、點餐、點數 & 折價券、訂閱方案、我的訂單、我的客訴 |

左側邊欄「首頁」群組可展開，包含：首頁儀表板、採購首頁、美編首頁、公告管理。「餐點管理」群組可展開，包含：餐點管理、餐點異動紀錄、店家異動審核。「菜單管理」群組可展開，包含：菜單排程管理、菜單發布管理、菜單管理異動紀錄、菜單模板管理。「問題單&ECR」同樣為可展開群組，子項目包含：📝 新增問題單、📋 問題清單、🔄 新增 ECR、📑 ECR 清單、📖 填寫指南。點擊子項目就地切換頁內 tab，無須重新載入頁面。店家後台左側邊欄包含：公告通知、餐點管理、訂單明細。

手機版（< 768px）：頂部列收合為 hamburger 選單，左側邊欄隱藏。

## 功能需求清單

> 各頁面皆須附有填寫指南，說明欄位規範與注意事項。
> 所有頁面須支援 RWD，針對手機版面進行優化。

### 首頁儀表板

- 統一各部門首頁框架，依角色顯示不同內容
- 角色切換下拉（讀取帳號管理的角色清單，fallback 為靜態預設清單；不含主管角色）
- **KPI 指標**（依角色預設，固定不可自訂；label 顯示於數值上方）：
    - Admin：全部指標
    - 採購：待審核異動、本週即將生效、本月有異動餐廳、本月新增品項
    - 美編：新店家、待審圖片、可製作 EDM、未讀公告
    - 營運：今日訂單量總計、銷售毛利總計、熱門店家（顯示名稱）、熱門餐點（顯示名稱）、待處理客訴
    - 業務：負責店家數、未讀公告、待處理客訴
    - 行銷：未讀公告、訂閱現況（各方案人數）、會員總點數、待使用折價券張數（動態從 sessionStorage 讀取）
    - 人資：未讀公告
    - IT：問題單通知、ECR 通知、未讀公告
    - 其他角色：未讀公告
- **常用功能**（依角色預設，使用者可透過「✦ 自訂版面」調整）：
    - 自訂結果儲存於 sessionStorage（`dashboard_config_v1`），按角色各自保存
    - 自訂 Modal：勾選要顯示的快捷連結、重置預設
    - 可勾選的項目依帳號管理的角色權限過濾（`permissions_roles_v2`），沒有對應權限的功能不出現在清單中；若尚未讀取到帳號管理資料，以各角色的靜態預設清單作為 fallback
- **公告區塊**：Tab 切換（全部 / 未讀 / 系統公告 / 活動資訊）；點擊公告列開啟詳細內容 Modal 並標記已讀；超過數筆顯示滾動條
    - 部分公告設有角色限制（`roles` 欄位），僅顯示給指定角色；無 `roles` 欄位則全角色可見
    - 採購：顯示「店家異動審核通知：待審項目」
    - 美編：顯示「店家異動審核通知：待審圖片」

### 美編首頁

- 三個待處理事項區塊，各含左側 KPI 數字卡（rgb(0,193,215)）與右側列表（超過 4 筆顯示滾動條）：
    - **新店家**：列出新進店家與品項數量、預計使用月份，可刪除通知；依預計使用月份正序排列
    - **待審圖片**：列出待審核圖片申請，顯示店家、數量、預計使用月份、申請日期；點擊任一列跳至「店家異動審核」並自動篩選圖片類；依申請日期正序排列
    - **可製作 EDM**：列出可製作的 EDM 通知，顯示年月、廠區、發布時間，可刪除通知；依發布時間正序排列
- 內部公告區塊：Tab 切換（全部 / 未讀 / 系統公告 / 活動資訊）；點擊公告列標記已讀（透明度降至 30%）；超過 4 筆顯示滾動條

### 公告管理

公告分類：系統公告、活動資訊、其他（可透過 ⚙ 管理分類新增自訂分類）。

- **公告通知（各系統接收端）**：
    - 查看全部公告，可依全部 / 未讀篩選
    - 標記已讀
    - 店家後台：僅顯示系統公告，使用列表式樣式（同首頁儀表板）
- **公告管理頁面（僅內部後台）**：
    - 新增公告，包含：標題、分類（下拉選擇）、顯示位置（複選：內部後台 / 店家後台 / 訂餐前台）、是否彈出通知、內容、圖片、附件
    - 可設定排程（發布時間、到期時間）
    - 公告狀態：發布中、草稿、排程待發、已到期
    - 搜尋標題、依分類 / 狀態 / 顯示位置篩選
    - 編輯、刪除公告
- **分類管理（⚙ 管理分類）**：新增自定義分類、點擊改名、刪除（有公告時阻擋）、拖移排序

### 餐點管理

- 可搜尋餐廳選擇器（旁顯示抽成 % chip，hover 顯示「來自客戶／供應商管理的合約設定」說明）
- 販售中 / 下架存檔區兩個 Tab
- 品項列表：料號、餐點名稱（含英文名稱、機台顯示名稱標注）、分類、店內價、終端價、成本價、摩寶連結狀態
- 店內價預約調整：有調升（紅 ↑）/ 調降（綠 ↓）顏色區分；成本價欄同步顯示「↕ 日期 自動更新為 $XX」
- 預約調整店內價：設定新店內價與生效日，系統於生效日自動套用並重算成本價，不需人工操作
- 終端價：可手動設定，與店內價獨立
- 清單 / 圖片切換檢視：
    - 清單模式：表格顯示，含料號、餐點名稱（英文名稱、機台顯示名稱）、分類、店內價、終端價、成本價、內部備註、摩寶狀態
    - 圖片模式：3 欄格狀卡片，顯示大圖、品名、英文名稱、機台顯示名稱、介紹（含欄位標籤）、三種價格、備註；hover 圖片區出現「上傳圖片」/「更新圖片」按鈕
- 圖片管理：
    - 單張上傳：於圖片模式卡片 hover 後點擊按鈕，或在編輯 modal 中上傳；編輯 modal 中可下載目前圖片
    - 批量上傳：選取多張圖片，檔名以品項名稱開頭（如 雙拼便當_1.jpg）自動對應品項
    - 批量下載：下載目前篩選結果中有圖片的品項，打包為 ZIP（示意）
    - 圖片放大檢視：圖片模式卡片圖、編輯 modal 縮圖均可點擊放大，支援 ESC 或點擊背景關閉
    - 疊加圖片設定：可勾選蛋奶素、僅供參考、截角圖（透明）、隨機出貨一款等標示圖疊加在餐點圖片上，套用後圖片模式所有有圖片的品項即時顯示疊加效果
- 新增 / 編輯品項：
    - 餐點完整名稱（無字數限制）
    - 機台顯示名稱（9 字上限，選填，留空自動截取）
    - 分類（下拉選擇，選項來自分類管理）
    - 排序（必填，新增時自動建議目前最大值 +1）
    - 英文名稱、介紹（顯示於訂餐前台）、內部備註（僅浩華人員可見）
    - 圖片上傳（單張，直接生效，無審核流程）
- 刪除品項（權限確認提示）
- 下架移至存檔區 / 重新上架
- 分類管理（⚙ 管理分類）：
    - 各餐廳獨立分類清單
    - 新增自定義分類、點擊改名、刪除（有品項時阻擋）
    - 拖移調整分類顯示順序
    - 儲存時自動同步品項的分類名稱
- 摩寶整合：
    - 未連結品項顯示「上傳摩寶」按鈕
    - 已連結品項顯示「已連結」badge、最後同步時間、「更新摩寶」按鈕（覆寫摩寶系統資料）
    - 圖片模式卡片底部同樣顯示摩寶狀態與操作按鈕
- 品項料號自動以現有最大號碼遞增，確保不重複
- Excel 匯入 / 匯出（示意）
- 分類篩選（顯示所有已定義分類）、搜尋品名或料號

### 餐點調價歷史紀錄

- 可搜尋餐廳選擇器
- 月份切換（上 / 下一月）
- 價格快照頁：
    - 顯示當月全部品項完整價格清單
    - 欄位：料號、餐點名稱、店內價、終端價、成本價（ℹ 成本價 ＝ 店內價 × (1 − 抽成%)）、異動標記
    - 異動標記：新增、異動、已下架、無異動
    - 統計數量（共幾筆、各標記數量）
- 月份快照異動備註自動產生：
    - 系統自動比對上月同料號資料，產生異動說明（品名異動、成本/售價調漲降、終端價異動、新增品項、已下架）
    - 多項異動以「・」連接，不需人工填寫
- 異動事件紀錄頁（流水帳）：
    - 依日期分組呈現每筆異動，顯示異動前後的值
    - 每筆紀錄包含：料號、餐點名稱、異動描述、日期、時間、操作人員、來源（人工操作 / 系統自動）
    - 依異動類型篩選（店內價異動、品名異動、上下架 / 新增）
    - 搜尋品名或料號

### 店家異動審核

- 可搜尋餐廳選擇器
- 待審核 / 已處理兩個 Tab
- 每筆異動顯示異動前後欄位比對（刪除線 vs 綠字）
- 店家須附上紙本菜單照片作為佐證
- 通知機制：任何異動申請送出後通知營運；審核通過或退回後亦通知營運
- 圖片類異動由美編審核，其餘（完整名稱 / 介紹 / 店內價）由採購審核
- 核准：可設定系統生效日期後確認寫入
- 退回：須填寫退回原因通知店家
- 篩選：依餐廳、異動類型（完整名稱 / 介紹 / 圖片 / 店內價）
- 菜單更新通知：店家可透過上傳實體菜單照片的方式申請異動，於審核端顯示為獨立通知類型

### 菜單排程管理

- 月曆導覽：點左右箭頭切換月份；點月份標籤開啟跳月視窗，可輸入年份 + 選月份或點快速捷徑直接跳至指定月；點「回到今日」回到當前月份
- 廠區下拉支援關鍵字搜尋，切換後月曆顯示對應廠區排程
- 餐期下拉可切換（全部 / 早餐 / 午餐 / 下午茶 / 晚餐 / 消夜）
- 新增店家：開啟後頂部顯示當前廠區、日期、餐期（唯讀）；由當前篩選餐期決定餐期（無需重複選擇）；可選擇套用模板（選填），套用後餐廳下拉僅顯示模板內的店家池，不套用則顯示全部餐廳；可搜尋選擇餐廳；若餐廳有分店則分店必填；勾選品項後展開設定欄（成本價唯讀顯示、終端價可修改、櫃位（溫度：熱櫃/冷櫃；格號：不指定/A櫃/B櫃/C櫃，各自獨立單選）、線上可預約數預設 30、線下可預約數、店家預定數，數量欄位獨立一行），支援全選 / 取消全選；品項清單支援即時關鍵字搜尋
- 月曆節日標註：台灣固定國定假日（元旦、和平紀念日、兒童節、勞動節、國慶日、行憲紀念日）於對應日期格子中顯示紅色小標籤，每年自動套用無需維護
- 月曆 Chip 快速預覽：滑鼠移到月曆格子中的店家 chip 或會議便當 chip 時，就地彈出提示視窗，顯示餐期（含顏色標示）、店家名稱、各品項名稱及終端價，無需捲頁即可瀏覽摘要
- 日期明細：點月曆格子展開當天所有排程卡片，品項顯示成本價（橘色）與終端價，可直接編輯各品項的終端價、櫃位（溫度與格號各自獨立單選）與數量（數量三欄獨立一行），支援移除品項 / 移除整間店家 / 追加品項
- 防呆規則：跨廠區同日**同餐期**同餐廳衝突偵測（灰色不可選 + 衝突提示）；不同餐期可使用同一餐廳（A廠午餐已排 → B廠晚餐仍可選同餐廳）；餐廳公休日不可排程（依餐期判斷）
- 發佈訂餐：選月份、複選廠區、複選餐期，按確認後跳出二次確認視窗顯示摘要，確定後送出並顯示成功通知
- 排程發布狀態：每間店家卡片顯示發布狀態（未發布 / 已發布 / 已修改），可個別發布 / 重新發布，或批量全部發布 / 更新已修改；發布後有修改可點「取消變更」還原至上次發布狀態（附二次確認）；若改回原始值狀態自動恢復為已發布
- 複製整月排程：一鍵將當前廠區當月所有排程（含所有餐廳、餐期）複製到其他據點，重複者自動略過；目標據點支援關鍵字搜尋複選
- 分店選擇：新增店家時，若餐廳有多家分店可進一步選擇分店，分店必填；店家排程卡片顯示分店（藍色 badge）、廠區名稱、發布日期
- 批次調整價格：toolbar 提供「批次調整價格」按鈕（整月）；詳細面板批量操作列另有同名小按鈕（當天）；支援固定金額（±元）或百分比（±%），套用前預覽原價與新價（含 ≤0 警告），套用後可一鍵還原
- 會議餐：月曆每格支援「新增會議餐」，開啟後頂部顯示當前廠區、日期、餐期（唯讀）；可填人員名稱（必填，未填送出自動捲動聚焦）、餐期、店家（有分店則必選）、分店、備註、提醒時間（依餐期自動預設：早餐 22:05、午餐 09:05、晚餐 13:05、消夜 18:05，可手動調整）、品項與數量（至少勾選一個，成本價唯讀顯示，終端價可手動修改，支援即時關鍵字搜尋）；同廠同時段可建立多筆不同人員的會議餐；月曆以粉紅色 chip 區分顯示；Detail Panel 以獨立卡片區塊呈現，品項顯示數量、成本價、應付帳款、終端價、應收金額、利潤，支援編輯
- 移除二次確認：移除店家排程、品項、會議便當時皆跳出確認對話框
- 匯出：支援 CSV 與 JSON 格式下載；CSV 新增「分店」欄位；JSON 按餐期分檔，每個有資料的餐期各產一個檔案，欄位含分店、據點、日期、星期、時段、英文名稱、餐廳名稱、餐廳編號、餐點名稱、餐點編號、售價、圖片網址；新增「會議餐匯出」功能，可選日期區間（跨月）與廠區，匯出 CSV 含人員名稱、品項、數量、成本價、應付帳款、售價、應收金額、利潤、備註、提醒時間
- 匯入：支援 CSV 與 JSON 格式上傳，自動對應廠區、餐廳、餐期、品項並寫入排程
- 販賣機操作按鈕：傳送訂單資料、傳送巡捕資料、重整餐點圖片（功能開發中）

### 問題單與 ECR 請求

- 通知機制：
    - 新問題單提交後，IT 工程師需收到通知
    - 問題單狀態更新或回填後，填寫人需收到通知
    - 新 ECR 提交後，主管需收到審核通知
    - ECR 審核結果出爐後，填寫人需收到通知
- 範本功能（可行的話）：
    - 問題單與 ECR 皆支援儲存為範本，下次可快速套用常用欄位

- 問題單 — 填寫區：
    - 必填欄位：
        - 部門、填寫人
        - 專案名稱、模組
        - 問題摘要
        - 嚴重程度（嚴重 / 高 / 中 / 低）
        - 問題類型（程式錯誤、資料錯誤、畫面異常、效能問題、版面問題、規格異動）
        - 哪一支網頁 / 程式、發生時間
        - 進行了什麼操作（操作步驟）
        - 發生了什麼問題（實際結果 vs 預期結果）
        - 以前有沒有發生過？
        - 其他人有沒有發生過？
        - 作業系統、瀏覽器
    - 選填欄位：裝置型號、正常環境資訊、重現步驟、測試資料、截圖 / 錄影附件
- 問題單 — 清單區：
    - 搜尋、嚴重度、狀態篩選
    - 統計數量顯示（全部、待處理、處理中、已解決）
    - 編輯、刪除問題單
    - IT 工程師回填：問題原因、修復方式
- ECR — 填寫區：
    - 必填欄位：
        - 專案名稱、模組
        - 申請日期、部門、填寫人
        - 變更標題
        - 變更類型
        - 優先級
        - 原因
        - 現行狀況（AS-IS）
        - 變更後期望狀況（TO-BE）
        - 影響的頁面 / 程式
        - 影響程度
        - 期望完成日期
    - 選填欄位：相關問題單編號、相依性、附件、備註
    - 主管進行審核（核准 / 退回 / 需討論）
- ECR — 清單區：
    - 搜尋、優先級、狀態篩選
    - 統計數量顯示（待審核、已核准、實施中）
    - ECR 狀態：草稿、待審核、審核中、已核准、已退回、實施中、已完成
    - 編輯、刪除 ECR
    - 主管回填審核結果（核准 / 退回 / 需討論）與審核意見
- ECR 變更類型管理（⚙ 管理類型）：新增自定義變更類型、點擊改名、刪除（有 ECR 時阻擋）、拖移排序

- **佈局**：左側為菜單網格，右側為固定 sticky 購物車（320px），無需捲動即可操作；行動版自動改為單欄
- **餐點卡片**：含圖片佔位區與數量選擇器（`− N +`），選好數量後點「加入購物車」，Toast 顯示「已加入購物車」
- **已訂閱會員**：直接顯示折扣後的訂閱價（原價 − 折抵金額），附「訂閱價」青色（teal）標籤
- **未訂閱會員**：顯示原價，並列出訂閱後的折扣參考價（省 $N 元），引導升級
- **訂閱提示橫幅**：頁面頂部依訂閱狀態顯示對應說明文字，未訂閱者提供「查看訂閱方案」連結
- **購物車**：支援數量調整、折價券選擇（勾選後即時折抵），結帳時折價券標記已使用
- **會員模擬切換器**：右上角下拉可切換模擬會員（無訂閱 / 有訂閱），即時反映不同訂閱狀態的價格顯示差異；切換後頁面間共享選擇（`sessionStorage.fe_sim_member`），跨頁不重置；F5 重整全部資料回到預設狀態（sessionStorage 清空）
- **預設折價券**：無訂閱會員預設持有一張 $10 新會員贈券

### 點數 & 折價券管理（內部後台）

後台點數與折價券管理頁（`coupon.html`），以雙 Tab 結構整合兩項功能，預設開啟點數管理。

**Tab 1：點數管理**

- **點數規則設定**：可自訂每消費 $N 元得 M 點（預設每 $100 得 1 點）、每 X 點可兌換 $Y 折價券（預設 100 點換 $10），修改後點擊儲存即時生效
- **節日促銷管理**：列表顯示所有促銷（名稱、類型、期間、啟用狀態）；可新增促銷，選擇「點數倍數」（輸入倍率）或「直接發折價券」（輸入金額）；可切換啟用 / 停用或刪除
- **會員點數查詢**：唯讀表格，顯示所有會員編號、姓名、持有點數；支援依姓名或會員編號搜尋

**Tab 2：折價券管理**

- **統計卡片**：已發放張數、待使用張數、已使用張數、待使用總折抵金額（金額數字以 teal 色顯示）
- **折價券列表**：顯示折價券碼、折抵金額、持有會員、來源（補償 / 促銷 / 推薦）、備註原因、有效期限、發放日期、狀態（未使用 / 已使用 / 已過期）
- **篩選功能**：可依關鍵字搜尋、來源類型、使用狀態篩選
- **新增折價券**：來源限「促銷」；補償類折價券僅能從客訴管理頁面發放；推薦折價券由系統於推薦成立後自動發放，不可手動新增。選擇持有會員、輸入金額與有效期限、填寫備註，系統自動產生 `COMP-XXXXX` 格式折價券碼
- **作廢操作**：可將未使用的折價券標記為已使用（作廢），附二次確認
